The Proposal Report is the main working screen in SiPRO module for reviewing current inventory settings against the module’s recommended replenishment values.
 
It is designed to help users assess stockholding decisions product by product, compare current and proposed values, and decide whether a recommendation should be accepted and applied.
 
Where to find it
 
To open the report:
 
1. Go to Inventory Risk Mitigation
2. Click Proposal Report
 
 
The report opens in a list view and displays one row per product.
 
Purpose of the Proposal Report
 
The Proposal Report helps users:
 
•review current stock and reorder settings
•understand the recommended reorder position for each product
•compare current stock value with proposed stock value
•identify products that may be overstocked, understocked, or incorrectly classified
•take action on proposals using the Accept, Apply, and Reset buttons
 
Workflow buttons
 
  
Accept
The Accept button marks the proposal as reviewed and accepted.
 
Use this when:
 
•the recommendation has been checked
•the user agrees with the proposed values
•the product is ready to move to the next step
Apply
The Apply button writes the accepted proposal into the product’s reordering rule.
 
Use this when:
 
•the proposal has already been accepted
•the user wants to update the live reorder values in Odoo
Reset
The Reset button returns an applied proposal back to draft status.
 
Use this when:
 
•you want to clear the proposal workflow status
•you want to review the item again before further action
 
 
Proposal Workflow Restriction
 
When a product has a Strategic Stock Type, proposal workflow actions in the Proposal Report are disabled for that product.
 
This is intentional. Strategic stock products are treated as exceptions and are not meant to be processed through the normal accept/apply proposal workflow.
 
The user may however still view-only the recommendations such as reorder point (i.e. minimum stock) and order up to (i.e. maximum stock) for strategic stock based on historical data and analysis.

Column
Description
How to Use It
Product Name
Displays the name of the product.
Use this as the primary product identifier in the report.
Internal Reference
Displays the internal product code or SKU.
Useful when products have similar names or when internal stock codes are used for operational review.
Current Min
Shows the current minimum quantity from the product’s existing reordering rule.
Represents the current reorder point in the live Odoo setup.
Current Max
Shows the current maximum quantity from the product’s existing reordering rule.
Represents the current maximum stock level in the live Odoo setup.
Reorder Point
Shows the proposed new minimum stock level calculated by the module.
Compare this with Current Min to see whether the recommendation suggests a higher or lower reorder threshold.
Order Up-To Point (Maximum Stock Level)
Shows the proposed target maximum stockholding level.
Compare this with Current Max to assess whether the current maximum is aligned with the proposed inventory policy.
Inventory Value Variance
Shows the difference between the proposed inventory value and the current stock value.
Provides a quick financial view of the impact of the recommendation. A positive variance may indicate increased inventory investment; a negative variance may indicate reduced inventory investment.
Product Category
Displays the product category assigned to the item.
Useful for reviewing products by category and identifying groups that may require similar policy decisions.
Vendor
Displays the main vendor used by the module for lead-time interpretation.
Derived from the first supplier line on the product template.
Strategic Stock Type
Displays whether the product has been marked as strategic or exceptional stock.
If this field contains a value, the product is being handled as an exception and normal proposal actions are disabled.
Current Stock / Non-Stock Status
Shows whether the product is currently treated as Stock or Non-Stock.
Helps users understand whether the product is being treated as a stocked item in the current operating setup.
Stock On Hand
Shows the current on-hand stock quantity.
Useful for comparing physical stock against current and proposed reorder levels.
Forward Order Qty
Shows the quantity already committed to outgoing demand.
Helps users understand future stock pressure that may not be obvious from stock on hand alone.
Current To Order
Shows the current order multiple from the product’s reordering rule.
Indicates the quantity step used when replenishment orders are placed.
Strategic Stock Holding Qty
Shows the strategic stock holding quantity displayed for proposal review.
Presented as a rounded whole-number display value.
Delivery Lead Time
Shows the supplier lead time used by the module, displayed in days.
Based on the first supplier record linked to the product. Lead time has a direct impact on replenishment calculations.
New Safety Stock
Shows the proposed safety stock calculated by the module.
Represents the recommended buffer stock to protect against demand variability and lead-time risk.
New Reorder Qty
Shows the proposed replenishment quantity.
This is the quantity the module recommends ordering once the reorder point is reached.
Current Stock Value
Shows the current stock on hand value based on the product’s standard cost.
Helps users understand the value of the stock currently being held.
New Min Stock Value (Reorder Point Value)
Shows the value of the proposed reorder point quantity.
Indicates the financial value of the proposed minimum stockholding level.
New Reorder Qty Value (Goods In Transit)
Shows the value of the proposed reorder quantity.
Represents the financial value of the replenishment quantity that would typically be in transit or purchased.
New Extended Inventory Value
Shows the combined proposed inventory value based on the proposed minimum stock value and the proposed reorder quantity value.
Gives a financial view of the total proposed inventory position.
Inventory Risk Mitigation
Shows the combined inventory classification code for the product, such as AX, BY, CZ, or ZZ.
This classification is one of the key outputs of the module and is used to drive service level and replenishment logic.
Variant Count
Shows the number of variants related to the product template.
Useful when reviewing products that may represent multiple variants under one template.
Unit Of Measure
Displays the unit of measure used for the product.
Helps users correctly interpret quantities shown in the report.
Unit Cost
Shows the product’s standard cost.
Used in the value-based classification logic and in the inventory value calculations shown in the report.
Safety Stock
Shows the proposal display value for safety stock.
This is the rounded report version intended for user-friendly review.
Reorder Point
Shows the proposal display value for reorder point.
This is the rounded report version intended for user-friendly review.
Reorder Frequency In Weeks
Shows the proposed replenishment frequency in weeks.
Based on the configuration matrix and the product’s classification and lead-time grouping.
 
How to use the Proposal Report
 
A practical review sequence is:
 
1. Identify the product using Product Name and Internal Reference.
2. Review the current position:
•Current Min
•Current Max
•Stock On Hand
•Forward Order Qty
•Current Stock Value
3. Review the business context:
•Product Category
•Vendor
•Strategic Stock Type
•Current Stock / Non-Stock Status
•Delivery Lead Time
4. Review the recommendation:
•New Safety Stock
•Reorder Point
•Order Up-To Point (Maximum Stock Level)
•New Reorder Qty
•Reorder Frequency In Weeks
5. Review the financial impact:
•Inventory Value Variance
•New Min Stock Value
•New Reorder Qty Value
•New Extended Inventory Value
6. Review the classification:
•Inventory Risk Mitigation
7. If appropriate, click Accept, then Apply.
 

 
 
Recommendation
 
•Review recommendations in batches by product category or vendor.
•Pay close attention to products with high inventory value variance.
•Check supplier lead time carefully before approving proposals.
•Use the report as a review and decision-making tool, not as a substitute for operational judgement.
•Validate a sample of recommendations before applying changes widely.
